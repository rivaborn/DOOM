# File Overview

**Path:** `sndserv/sounds.h`
**RCS:** `$Id: sounds.h,v 1.3 1997/01/29 22:40:44 b1 Exp $`

`sounds.h` is the master enumeration header for all audio content (both sound effects and music tracks) in DOOM. It was originally generated by Dave Taylor's sound utility and kept as a canonical reference. It defines the complete integer ID sets for every SFX and music cue in the game.

This file is shared in concept with the main engine (`linuxdoom-1.10/sounds.h`), but the copy in `sndserv/` is used exclusively by the sound server process. It provides:

1. `musicenum_t` — an enumeration of every music track.
2. `sfxenum_t` — an enumeration of every sound effect.
3. `extern` declarations for the master data arrays `S_music[]` and `S_sfx[]` (defined in `sndserv/sounds.c`).

---

## Global Variables

Declared as `extern` (defined in `sndserv/sounds.c`):

| Type | Name | Purpose |
|---|---|---|
| `musicinfo_t[]` | `S_music` | Array of `musicinfo_t` structs indexed by `musicenum_t`. Contains names, lump numbers, data pointers, and handles for all music tracks. |
| `sfxinfo_t[]` | `S_sfx` | Array of `sfxinfo_t` structs indexed by `sfxenum_t`. Contains names, priorities, link pointers, and data pointers for all sound effects. This is the primary table used by `soundsrv.c` during initialization and playback. |

## Enumerations

### `musicenum_t`

An enumeration of all music tracks in DOOM and DOOM II. The value `NUMMUSIC` (the last enumerator) gives the total count.

| Range | Tracks |
|---|---|
| `mus_None` (0) | Null/no-music sentinel |
| `mus_e1m1` – `mus_e1m9` | DOOM Episode 1 level music |
| `mus_e2m1` – `mus_e2m9` | DOOM Episode 2 level music |
| `mus_e3m1` – `mus_e3m9` | DOOM Episode 3 level music |
| `mus_inter`, `mus_intro`, `mus_bunny`, `mus_victor`, `mus_introa` | DOOM Ep. intermission, intro, bunny screen, victory, shareware intro |
| `mus_runnin` – `mus_dm2int` | DOOM II level and screen music |
| `NUMMUSIC` | Total count of music entries |

## Dependencies

|---|---|

| File | Why |
